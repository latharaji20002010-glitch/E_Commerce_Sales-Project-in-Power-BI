# E_Commerce_Sales-Project-in-Power-BI
# Description
    As a Data Analyst, I have to  work with an E-commerce sales dataset to perform data preparation and modeling using Power BI. The dataset comprises multiple related files that represent order-level, product-level, and sales target information.
    
# Objective
    I have developed interactive visualizations to analyze sales trends, profitability, and performance indicators within the  e-commerce business environment by data import, transformation, cleansing, aggregation, and relationship modelling using Power BI and Power Query Editor, DAX calculations, analytical thinking, and visual storytelling,  while ensuring that the underlying data model supports accurate reporting and insights, which is used to improve the sales operations.
    
# Data Source
    [List of Orders.csv](https://github.com/user-attachments/files/29405027/List.of.Orders.csv)
    [Order Details.csv](https://github.com/user-attachments/files/29405028/Order.Details.csv)
    [Sales target.csv](https://github.com/user-attachments/files/29405029/Sales.target.csv)

# Data Transformation
    The Power Query editor is used to transform data, including row limiting, changing data types, Text Formatting, Merging columns, creating columns, Conditional Column Creation, Merging Tables, handling missing data and duplicate data, Sorting and Filtering Data, and Grouping and Aggregating Data.
    
# Data Model & DAX
    Snowflake schema Data Modeling is used. List of Orders is a primary fact table. Order details, Sales target, Orders data, and Date tables are Dimension tables.   DAX Calculations are used to create calculated columns and Calculated Measures.

# Dashboard
    Revenue vs Target, Quantity, and Profit by Sub category, Order Count, Year-to-date Sales, Average Profit in Delhi, Category-wise Monthly Revenue vs Target, and Monthly Sales Trend were shown in the interactive dashboard with filters.
